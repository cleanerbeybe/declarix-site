# SEO and CRO rationale

## Search job

The route targets buyers trying to quantify customs declaration cost, customs clerk capacity, labour
per declaration, or a software business case. It is useful before the user knows Declarix and close
enough to the commercial problem to produce warm discovery-call traffic.

## On-page contract

- Exact descriptive title and URL.
- One outcome-led H1.
- A direct-answer paragraph and copyable formula high on the page.
- Complete static HTML for crawlability; JavaScript enhances only the arithmetic.
- Canonical, description, social metadata, sitemap entry, `llms` entries, breadcrumb markup,
  `WebApplication`, and visible FAQ content with matching `FAQPage` schema.
- Internal navigation links from every generated static route.

## Conversion sequence

1. Promise a buyer-controlled result with no sign-up.
2. Answer the core cost formula before asking for interaction.
3. Give immediate example outputs and invite replacement with real desk data.
4. Lead the result with annual labour headroom and break-even per entry.
5. Make formula and assumptions inspectable.
6. Offer share and print for an internal buying discussion.
7. Ask for the existing 20-minute numbers call to validate workflow, integration, and rate.

The existing homepage, pricing copy, and booking UI are not edited. This route adds a qualified path
into them instead of substituting a generic lead-capture gate.
